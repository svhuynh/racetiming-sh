Methods

(inner) get/:passcode()

Checks if the passcode matches a passcode in database. Can be matched to 4 roles: admin, timer, organizer, and announcer. They correspond respectively to roles 1, 2, 3, and 4. A token is emitted after authentification. This token is verified for most API calls.
Parameters:
Name Type Description
req.params.passcode String passcode that was entered by the user

returns HTTP response code 200 on success with role and token for authentification. Otherwise, returns 500 with an error message.
